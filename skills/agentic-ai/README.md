# Agentic AI Hub (`agentic-ai`)

**自律型 AI エージェントの設計・運用を支援する、マルチファイル構成のスキルセットです。**

## 🎯 概要

`agentic-ai` は、Microsoft の **GH-600 (Developing in Agentic AI Systems)** 認定ガイドの理念をベースに、AI エージェントが人間の「意識の外」で自律的に満点の成果を出すためのプロトコルを提供します。

単一の指示書ではなく、計画・実行・評価・記憶の各フェーズを分離した複数のサブスキルで構成されており、複雑なプロジェクトの SDLC（ソフトウェア開発ライフサイクル）を AI が主導できるように設計されています。

## 🏗️ アーキテクチャ（三層構造）

本スキルセットは、エージェントの知能を以下の三層に構造化して管理することを推奨します。

1. **Layer 1: Global Law (共通の掟)**
   - ワークスペース全域で守るべき「憲法」。
   - セキュリティ、コスト監視、パブリック・ブランディング等の永続的制約。
2. **Layer 2: Task Specification (タスク個別仕様)**
   - 実装前に人間と合意する「設計図」。
   - ゴール、成功基準（DoD）、リスクの明文化。
3. **Layer 3: Project Context (暗黙の了解)**
   - プロジェクト特有の慣習や過去の決定事項。
   - `oracle.md` 等への蓄積と参照。

## 📂 構成ファイル

- **SKILL.md**: ハブ・エントリポイント。状況のヒアリングと Stage 判定、各サブスキルへのルーティングを担う。
- **01-architecture.md**: 計画 (Planning) フェーズ。Layer 2 仕様書の生成と成功基準の定義。
- **02-tooling.md**: 環境設計。ツール、権限、制約、および Layer 1 の生成。
- **03-memory-state.md**: 状態管理。長時間実行におけるコンテキストドリフトの防止と成果物の永続化。
- **04-evaluation.md**: 評価と調整。エラー分析、根本原因特定、プロンプト/ツールのチューニング。
- **05-multi-agent.md**: 協調設計。複数の専門知能をオーケストレーションするパターンの定義。
- **06-guardrails.md**: 安全策。人間による介入（Human-in-the-loop）と責任ある AI 原則の適用。

## 🚀 使い方

AI エージェントに対し、ハブとなる `SKILL.md` の URL を渡してください：

> 「`https://skills.dencat.dev/skills/agentic-ai/SKILL.md` を読み込んで、新しいタスクの設計を開始してください。」
