# 01-architecture: エージェント着手前準備プロトコル

## このSkillの目的

AIエージェントに作業を任せる前に「失敗したときのコストを確定させる」ことが目的。  
ヒアリング → リスク評価 → ガード設計 → Stage判定 の順に進める。

---

## Section 1: ゴールヒアリング

以下を順に確認する。ユーザーが答えられない項目は一緒に決める。

```
Q1. 最終的に何が完了した状態をゴールとするか？
    （例：PRが作成されている／テストが全部通っている／ファイルが変換されている）

Q2. 成功・失敗をどう判定するか？
    （例：exitコードが0／特定ファイルが存在する／出力に特定文字列が含まれる）

Q3. 制約・禁止事項はあるか？
    （例：本番環境には触らない／特定ブランチには直接pushしない）

Q4. 期限や実行回数の制約はあるか？
    （例：1回限り／毎日定期実行／CI上で動かす）
```

**ヒアリング完了の条件**：Q1〜Q4が全て答えられている状態。  
不明点があれば先に進まない。

---

## Section 2: 操作対象のリスク評価

エージェントが触るリソースを列挙し、リスクレベルを判定する。

### リスクレベル判定表

| リスクレベル | 条件 | 例 |
|---|---|---|
| **低** | 読み取り専用 / 一時ファイル / 再生成可能 | ログ読み取り、tmpファイル生成、テスト実行 |
| **中** | 書き込みあり / 復元可能 | ソースコード変更、設定ファイル更新、DBのCRUD |
| **高** | 削除・上書き / 本番影響 / 外部送信 | 本番DB更新、ファイル削除、外部API呼び出し、メール送信 |

### 判定フロー

```
操作対象を列挙する
  ↓
各対象に対してリスクレベルを判定する
  ↓
最も高いリスクレベルをタスク全体のリスクレベルとする
```

---

## Section 3: ガード設計

リスクレベルに応じてガードを決定する。

### 低リスク
```
- ガード不要
- そのまま実行許可
```

### 中リスク
```
- 以下のいずれかを必ず実施してから実行する
  □ git commitで現状を記録する
  □ 対象ファイルのバックアップを作成する
  □ dry-runオプションで事前確認する

- 実施するガードをユーザーと合意してから進む
```

### 高リスク
```
- 以下を全て実施する
  □ 操作内容をユーザーに提示し、明示的な承認を得る
  □ git管理またはバックアップで復元可能な状態にする
  □ 実行前に影響範囲を列挙して確認する
  □ ロールバック手順を決めておく

- 承認なしに実行してはならない
```

### ガード設計の記録

```
タスク名：[タスク名]
リスクレベル：[低/中/高]
実施するガード：
  - [ガード内容1]
  - [ガード内容2]
承認者：[ユーザー名 or 自動承認]
```

---

## Section 4: Stage判定

タスクの複雑度に応じてStageを決定し、読み込む後続Skillを決める。

```
判定フロー：

Step1. タスクは単一の操作で完結するか？
  Yes → Stage 1（→ 02-tooling.mdを読む）
  No  → Step2へ

Step2. 複数ステップがあり、前のステップの結果を次に引き継ぐ必要があるか？
  Yes → Stage 2（→ 02-tooling.md + 03-memory-state.mdを読む）
  No  → Step3へ

Step3. 複数のエージェントが並列・協調して動く必要があるか？
  Yes → Stage 3（→ 02-tooling.md + 03-memory-state.md + 05-multi-agent.mdを読む）
```

**注意**：04-evaluation.mdと06-guardrails.mdは全Stageで常に参照する。

## Section 5: 作業仕様の出力と保持

### 通常時
Section 1〜4の結果をトークンとして保持し、後続Skillにそのまま渡す。

### ファイル化するタイミング
以下のいずれかに該当する場合、`./copilot/`に作業仕様ファイルを作成する。

```
- Stage 2 または Stage 3 に進む場合
- 会話が長期化しそうな場合（目安：10ターン以上）
- マルチセッションにまたがる作業の場合
- 高リスクタスクで承認記録を残す必要がある場合
```

### 出力フォーマット

**ファイルパス**：`./copilot/{task-name}-spec.md`

```markdown
# {task-name} 作業仕様

## ゴール
{Q1の回答}

## 成功判定基準
{Q2の回答}

## 制約・禁止事項
{Q3の回答}

## 操作対象とリスクレベル
| 対象 | リスクレベル |
|---|---|
| {対象1} | {レベル} |

## 実施するガード
- {ガード内容}

## Stage
Stage {1/2/3}

## 読み込む後続Skill
- 02-tooling.md
- （03-memory-state.md）
- （05-multi-agent.md）
- 04-evaluation.md（常時）
- 06-guardrails.md（常時）

## 承認記録
承認者：{名前}
承認日時：{日時}
```

## 完了チェックリスト

このSkillを終える前に以下を確認する。

```
□ Q1〜Q4が全て回答済み
□ 操作対象のリスクレベルが判定済み
□ リスクレベルに応じたガードが決定・合意済み
□ Stageが判定済み
□ 必要に応じて./copilot/に仕様ファイルが作成済み
□ 後続Skillへの引き継ぎ情報が準備済み
```

全てチェックできたら後続Skillに進む。
